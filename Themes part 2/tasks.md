## Общие требования к коду

### Переменные
```python
user_score = 100  # хорошо
a = 100           # плохо
```

### Отступы
```python
if x > 0:
    print(x)  # 4 пробела
```

### Форматирование
- Пробелы после запятых: `print(a, b)`
- Пробелы при бинарных операциях: `x = a + b`
- Нет пробелов перед скобками функций: `print()` ✅, `print ()` ❌

### Блоки `if-elif-else`
```python
if a > 1:
    b = 3
else:
    b = 5
```

### Запрещено
- Дублирование кода
- Огромные функции
- Плохой нейминг
- Использование имён встроенных функций как переменных

### Общие критерии оценки
- Использованы именованные индексы, не просто `i`.
- Правильно оформлен `input`, без пустого приветствия для ввода.
- Переменные имеют значащие имена, не только `a, b, c, d`.
- Есть пробелы после запятых, пробелы при бинарных операциях.
- Нет пробелов после имён функций и перед скобками: `print()` — верно, `print ()` — неверно.
- Переменные имеют корректные названия, не используются имена встроенных функций.
